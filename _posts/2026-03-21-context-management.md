### 一、LLM 的记忆痛点：Agent 记忆模块的研发动因

LLM 本质为无状态模型，存在四大核心记忆局限，无法满足 Agent 的智能需求：

1. **上下文窗口限制**：超出窗口的信息会被遗忘，无法跨长序列获取信息；
2. **复杂任务处理乏力**：难以追踪多轮对话 / 子任务的状态和决策，交互轮数增加时连贯性缺失；
3. **无个性化能力**：无法记住用户偏好和历史互动，每次交互均为 “首次接触”；
4. **长上下文的性能与成本问题**：推理速度变慢、关键信息检索能力下降，且 Token 费用随上下文长度大幅增加。

而记忆模块能为 Agent 赋予**长期信息管理、知识持续更新、个性化服务、复杂任务支撑、交互质量提升**五大核心能力，成为 Agentic AI 的关键基础设施。

### 二、Agent 记忆的核心分类：短期 + 长期双层架构

智能体的记忆系统分为两类，适配不同场景的信息存储需求，二者协同实现完整记忆能力：

1. **短期记忆（STM / 工作记忆）**
    
    维护当前对话和任务的即时上下文，含**会话缓冲记忆**（最近对话滚动窗口）和**工作记忆**（任务临时中间结果 / 变量），受限于 LLM 上下文窗口，适用于简单对话 / 单一任务。
2. **长期记忆（LTM）**
    
    跨会话、跨任务持久化存储知识，依赖外部存储实现，主要形式有**摘要记忆**（长对话关键信息提炼）、**结构化知识库**（数据库 / 知识图谱存储）、**向量化存储**（向量数据库实现语义检索），适用于知识密集型应用和长期个性化场景。

### 三、Agent 记忆系统的设计与实现：四大核心环节

开发记忆系统需围绕**记忆产生、记忆策略、记忆存储、记忆检索**展开，同时结合**上下文工程**实现记忆的高效利用：

1. **记忆产生**：按场景筛选关键信息，如代码助手记项目架构 / 编码风格、智能客服记用户问题 / 解决方案、个人助理记日程 / 行为偏好、推荐 Agent 记用户显式 / 隐式反馈；
2. **记忆策略**：支持**轮数触发**（3-5 轮对话自动生成摘要）、**事件触发**（任务完成 / 场景转换时记录），也允许用户主动标记 / 删除记忆，保障数据控制权；
3. **记忆存储**：采用**用户→会话→记忆片段**三层结构，搭配元数据（时间 / 关键词 / 来源），复杂系统可拆分多个记忆库（短期工作记忆、长期情节记忆等）；
4. **记忆检索**：通过**关键词匹配、向量语义搜索、元数据过滤**检索相关记忆，按相关度排序后注入 LLM 上下文；
5. **上下文工程**：与记忆系统形成共生关系，作为记忆的 “调度员”，通过**检索生成、长序列处理、记忆层次管理**三大组件，从记忆库中筛选并组织最优信息给 LLM，还可通过文档分块、动态调取、上下文释放等策略解决超 Token 限制问题。

### 四、主流 Agent 记忆方案：开源框架 + 亚马逊托管服务

文章分析了 3 款主流开源记忆框架（与亚马逊云科技生态深度集成），并推出**Bedrock AgentCore Memory**托管方案，覆盖从定制化开发到开箱即用的全需求：

1. **Mem0**
    
    专为 Agent 设计的开源框架，支持多类型记忆（工作 / 事实 / 情景 / 语义）和多模态 / Graph 记忆，采用**六层架构**和**双 LLM、智能去重、冲突解决**等核心创新，可直接调用接口或封装为工具集成，与 Amazon Bedrock/OpenSearch/Neptune Analytics 等无缝对接，适合企业级深度定制；
2. **Letta（原 MemGPT）**
    
    借鉴操作系统 “虚拟内存” 概念，采用**上下文内 / 外双层记忆架构**，窗口满时自动压缩对话为递归摘要，支持记忆编辑与检索，可基于亚马逊云科技搭建无服务器架构，适合电商客服等长期交互场景；
3. **LangMem（LangChain 开发）**
    
    借鉴人类心理学记忆分类，设计**语义 / 情节 / 程序**三大核心记忆，支持跨会话知识连续性，与 LangGraph 深度集成，内置轻量存储适配开发测试，适合快速构建有学习能力的动态 Agent；
4. **Amazon Bedrock AgentCore Memory（托管方案）**
    
    开箱即用的持久化记忆服务，无需运维底层资源，提供**短期 + 长期**双层记忆，内置**语义 / 摘要 / 用户偏好**三大记忆策略，也支持自定义策略；提供检索 API 和工具化调用能力，数据加密存储且按命名空间隔离，实现记忆的自动提取、检索和上下文注入，大幅降低 Agent 记忆功能的开发成本。

### 五、核心结论

记忆机制是 Agentic AI 实现**认知连续性**的核心，是智能体从 “反应式” 向 “主动式 / 学习式” 进化的关键，而非可选功能。无论是基于开源框架（Mem0/Letta/LangMem）结合亚马逊云科技生态做定制化开发，还是直接使用 Bedrock AgentCore Memory 托管方案，都需将记忆设计融入 Agent 的核心架构；而亚马逊云科技则通过 “开源框架集成 + 托管服务” 的双重模式，为开发者提供了从技术验证到企业级部署的全流程 Agent 记忆解决方案。
